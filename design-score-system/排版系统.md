# Travai 白底版排版系统

版本：2026-07-17 01:00

这份文档是白底版唯一的排版基准。后续新增页面或调整组件，应优先复用这里的规则，不单独创造新的字号或字距。

## 1. 字体角色

- **品牌 Logo**：使用品牌图片资源，不用 CSS 重新绘制字标。
- **主标题**：`Avenir Next`，回退到 `Trebuchet MS` 和本地 `Travai Sans`。它负责理性、清晰的产品气质，与 Logo 的手绘情绪形成对比，不直接模仿 Logo。
- **正文与界面英文**：优先使用本地 `Travai Sans`，实际字体文件为 `assets/fonts/Inter-Variable.ttf`。
- **中文**：优先使用系统中文字体 `Noto Sans SC`、`PingFang SC`，保证字形稳定和阅读清晰。
- **等宽信息**：仅用于代码、技术信息或需要固定字符宽度的内容。

## 2. 字号层级

| 层级 | CSS token | 当前规格 | 用途 |
| --- | --- | --- | --- |
| Display | `--type-display` | `clamp(48px, 5vw, 64px)` | 主标题 |
| Section | `--type-section` | `22px` | 区块标题、侧栏主要导航 |
| Body | `--type-body` | `15px` | 正文、输入提示、历史记录 |
| UI | `--type-ui` | `13px` | 标签、辅助说明、规划信号 |
| Brand tag | 固定值 | `16px / 700` | Logo 下方中文标语 |

字号只允许在这几个层级内变化。需要强调时优先使用颜色、字重或留白，不额外增加新的字号。

## 3. 行高与字距

- 主标题：`line-height: 1.08`，保持两行紧凑但不粘连。
- 正文：`line-height: 1.62`，优先保证中文长句的呼吸感。
- 界面文字：`line-height: 1.35`，适合按钮、导航和短标签。
- 全站默认 `letter-spacing: 0`。不使用人为拉宽的英文字符间距。
- 英文短标签不得通过增加字距制造“高级感”；通过字号、字重和周围留白完成层级。

## 4. 混排基线

- 中文和英文放在同一行时，使用统一的 UI 行高，不给英文单独增加上、下偏移。
- 横向规划信号使用 `align-items: baseline`，确保“天气异常提醒”和“越来越懂你的 AI”落在同一条文字基线上。
- 分隔点使用统一的左右 `10px` 间距，不能让最后一个绿色信号脱离整行节奏。
- 绿色强调文字保持 `inline-block`，只改变颜色，不改变字号或垂直位置。

## 5. 组件文字

- 侧栏导航：正文级别，中文与英文共享同一行高；选中项使用品牌绿色，不改变字号。
- 搜索框提示：正文级别，颜色降低，不使用斜体或额外字距。
- 搜索按钮：UI 级别，字重 `700`；主按钮用品牌绿色，次按钮保持中性边框。
- Logo 标语：`16px / 700 / 1.25`，与 Logo 视觉中心对齐，和 Logo 保持紧邻关系。

## 6. 响应式规则

- 主标题只在 `48px` 到 `64px` 范围内流动，不让标题无限放大。
- 正文和 UI 文字保持固定层级，移动端优先通过换行和容器宽度解决拥挤。
- 不使用负字距，不使用依赖浏览器缩放的文字定位。
- 调整布局时先检查文字是否溢出、遮挡或产生新的基线错位，再调整间距。

## 7. 修改检查清单

1. 是否复用了现有四级字号，而不是新增字号？
2. 中文、英文和绿色强调文字是否位于同一基线？
3. 英文是否仍然保持 `letter-spacing: 0`？
4. 标题、正文、按钮和侧栏是否有清晰且稳定的层级？
5. 桌面和移动端是否都没有溢出、遮挡或因字体变化导致的布局跳动？
